# 00 - Registro de Evidências e Validações
## Projeto: VitahAcre

---

## 1. Objetivo

Este arquivo é o registro oficial de evidências do projeto.

No Método Caracol, não basta implementar. É preciso **provar o que foi feito**.

Cada ciclo deve depositar aqui seus registros de validação antes de ser considerado concluído.

---

## 2. Formato de Registro

Cada entrada deve conter:

```
### [Ciclo N] — [Nome do Ciclo]
- Data: YYYY-MM-DD
- Tipo: print / log / checklist / relatório / build
- Descrição: o que foi validado
- Arquivo(s): nome do arquivo de evidência nesta pasta
- Status: aprovado / reprovado / parcial
```
